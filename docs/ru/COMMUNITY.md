# Комьюнити

Этот репозиторий публичный и развивается вместе с комьюнити.

## Где взаимодействовать

- **Discussions** — архитектура, roadmap и вопросы эксплуатации
- **Issues** — воспроизводимые баги и конкретные feature-запросы
- **Pull requests** — точечные изменения кода и документации с зелёными проверками

## Какие отчеты полезнее всего

Сильный отчёт обычно содержит:

- точную команду запуска
- детали окружения
- очищенные от секретов логи
- ожидаемое и фактическое поведение
- указание, где именно проявилась проблема:
  - minimal xhttp-first install
  - `install --advanced`
  - `migrate-stealth`
  - клиентский вариант `recommended` или `rescue`

## Особенно полезные темы полевой обратной связи

- живучесть xhttp в реальных сетях
- поведение `packet-up` rescue на сложных провайдерах
- качество миграции с legacy `grpc/http2`
- совместимость артефактов в v2rayn, nekoray и raw xray клиентах
- поведение rollback и repair после ошибок

## Чего лучше не делать

- отправлять скриншоты без текстовых логов
- писать "`не работает`" без деталей
- публиковать приватные ключи, `keys.txt` или чувствительные ссылки
- смешивать несколько несвязанных багов в одном issue

## Ожидания к PR

- одно понятное изменение на один PR
- тесты и docs обновлены в том же проходе
- rollback и security-контракты не сломаны
- CI полностью зелёный до review
