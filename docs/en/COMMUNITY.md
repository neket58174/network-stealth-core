# Community

This repository is public and community-driven.

## Where to collaborate

- **Discussions**: architecture topics, usage questions, roadmap feedback
- **Issues**: reproducible bugs and concrete feature requests
- **Pull requests**: focused code/docs changes with passing checks

## Good issue format

Include:

- exact command used
- environment details (OS, shell, virtualization context)
- sanitized logs
- expected vs actual behavior
- minimal reproduction steps

Avoid:

- screenshots without text logs
- private secrets (`keys.txt`, raw private keys, full sensitive links)
- broad "does not work" reports without context

## Good PR format

- one problem per PR
- concise commit messages
- docs and tests updated together with behavior changes
- green CI before review request

See [.github/CONTRIBUTING.md](../../.github/CONTRIBUTING.md) for full contributor policy.

## Discussion topics that help the project

- reproducible installation edge cases
- profile and domain planning behavior under real-world load
- client compatibility improvements (v2rayN, NekoRay, Sing-box, ClashMeta)
- operational reliability and rollback quality
