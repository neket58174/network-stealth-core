# FAQ

## Is this project production-ready?

The project is a public automation script with CI and release gates.  
Use it with your own operational responsibility and host hardening standards.

## Which OS is officially supported?

Current validated platform:

- Ubuntu 24.04 LTS

Other Linux distributions may work but are not under active CI contract.

## Why are all destination ports shown as `:443`?

This is expected. Port `443` is standard HTTPS and usually the least suspicious path.
Listening ports for your inbounds are separate and generated by the script.

## What does `fallback on domain` mean in logs?

Example: `SNI img.example.com unavailable; fallback to example.com`.

It means the preferred SNI endpoint did not pass availability checks, so planner selected another valid SNI for the same destination domain.

## Can I create mixed profile sets (for example RU + one global)?

Default `install` now uses the minimal xhttp-first path (`ru-auto`).
Use `install --advanced` or explicit profile overrides when you need manual profile selection (`ru`, `global-50`, or `custom`).
Mixed pools in one pass are not first-class yet; use separate generation actions.

## Why does minisign prompt appear?

When release does not expose a minisign artifact or verifier is unavailable, script can ask whether to continue with SHA256-only mode.
Use `--require-minisign` for strict fail-closed behavior.

## Is this tied to one person or one server?

No. Repository content, docs, and defaults are intended for public generic usage.
